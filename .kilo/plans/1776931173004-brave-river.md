# Plano de Correção - Know You Work & Avaliação de Clientes

## Problemas Identificados

1. **Erro "Unexpected token '<'" nos relatórios de Colaboradores**
   - Solicitação para `/api/reports/collaboration/360-overall` retorna HTML (provavelmente página de erro ou 404 do Vite) em vez de JSON.
   - Causas possíveis:
     - Rotas de relatórios de colaboração não estão registradas corretamente (mas foram adicionadas na linha 2439+)
     - Queries das rotas falham porque tabelas `employees` ou `collaboration_forms` não existem
     - Erro não tratado retorna HTML via Express error handler ou Vite fallback

2. **"Nenhum formulário 360° configurado" na Avaliação 360°**
   - GET `/api/collaboration/forms?form_type=360` retorna array vazio.
   - O seed do formulário 360° pode não ter rodado (require restart do servidor)
   - Ou o seed falhou silenciosamente (ex: FK constraint em `created_by=1`)

3. **Links de avaliação de clientes podem não estar funcionando**

## Ações Corretivas

### A. Reiniciar o Servidor (Passo Obrigatório)
- Parar o servidor e iniciar novamente para executar:
  - Criação das tabelas `employees` e `evaluation_links` (blocos `checkCollabTable`)
  - Criação/seed do formulário 360° padrão
  - Migrations de colunas

### B. Verificação Manual Pós-Reinício
1. Verificar console do servidor: deve mostrar:
   - `✓ Created employees table`
   - `✓ Created evaluation_links table`
   - `✓ Seeded default 360° evaluation form`
2. Verificar no banco (toknow.db) se a tabela `collaboration_forms` contém um registro com `form_type = '360'`

### C. Tratamento de Erros nas Rotas de Relatórios (Backend)
Adicionar try-catch em todas as rotas de `/api/reports/collaboration/:type`:
- Retornar `res.status(500).json({ error: err.message, query: type })` em vez de deixar propagar
- Garantir que erros de SQL (como "no such table") retornem JSON

### D. Robustez no Seed do Formulário 360°
- Buscar `created_by` dinamicamente:
  ```javascript
  let createdBy = 1;
  const admin = db.prepare("SELECT id FROM users WHERE role='Administrator' LIMIT 1").get();
  if (admin) createdBy = admin.id;
  ```
- Se a tabela `users` estiver vazia, usar `null` (coluna permite NULL)

### E. Testes de Validação
1. Testar diretamente no navegador ou curl:
   - GET `/api/reports/collaboration/360-overall` → deve retornar JSON array (vazio ou com dados)
   - GET `/api/collaboration/forms?form_type=360` → deve retornar array com 1+ formulários
2. No frontend:
   - Acessar `/relatorios` → aba "Colaboradores" → deve carregar sem erro
   - Acessar `/avaliacoes/360` → deve mostrar formulário 360° disponível

### F. Se o Formulário 360° Ainda Não Existir
- Criar manualmente via POST `/api/collaboration/forms` com:
  ```json
  { "title": "Avaliação 360° - Know You Work", "description": "...", "form_type": "360", "entity_type": "Employee" }
  ```
- Em seguida, adicionar questões via POST `/api/collaboration/forms/{id}/questions` (pesar 2.0, 1.5, etc.)

## Risco
- Baixo: as mudanças são locais ao código e banco
- Reinício do servidor necessário para refletir mudanças no código

## Report
- Após correções, todos os endpoints devem retornar JSON válido
- Interface de Avaliação 360° deve mostrar formulário
- Relatórios de Colaboradores devem renderizar gráficos/lista
