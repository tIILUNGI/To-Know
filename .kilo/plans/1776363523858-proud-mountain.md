# Plano de Redesenho Responsivo - TO KNOW

## Objetivo
Melhorar a responsividade do sistema para funcionar bem em todos os tamanhos de ecrã, com especial atenção ao laptop. Simplificar a visualização mantendo todas as funcionalidades.

---

## Abordagem

- **Navegação**: Manter menu no topo (atual)
- **Visual**: Simplificar componentes, melhor organização
- **Foco**: Ecrãs de laptop (1366px - 1920px)

---

## 1. Layout e Estrutura Global

### 1.1 Layout Principal (`Layout.tsx`)
- Reduzir padding interno do `main` de `p-6` para `p-4 sm:p-6`
- Limitar largura máxima do conteúdo a `max-w-7xl mx-auto`
- Ajustar altura do header para `h-14` (mais compacto)
- Melhorar espaçamento entre nav items

### 1.2 CSS Global (`index.css`)
- Adicionar `container` class responsiva com centered margins
- Melhorar breakpoints de responsividade
- Simplificar estilos de tabelas para mobile
- Ajustar tipografia para melhor legibilidade

---

## 2. Páginas Específicas

### 2.1 Dashboard (`Dashboard.tsx`)
- **Filtros**: Empilhamento vertical em mobile, linha única em desktop
- **KPI Cards**: Grid 2 colunas em tablet, 4 colunas em desktop
- **Gráficos**: Altura adaptativa, ocultar legendas em mobile
- **Gauge Cards**: Empilhar vertical em mobile, lado a lado em tablet+

### 2.2 Lista de Entidades (`EntityList.tsx`)
- **Tabela**: Scroll horizontal em mobile com `overflow-x-auto`
- **Filtros**: Campo de pesquisa + selects em grid flexível
- **Ações**: Botões de ação mais acessíveis em mobile
- **Cards**: Alternar para visualização em cards em mobile

### 2.3 Processo de Aprovação (`ProcessWorkflow.tsx`)
- **Stats bar**: Horizontal scroll em mobile
- **Cards de Processo**: 1 coluna mobile, 2 tablet, 3 desktop
- **Filtro de estado**: Select dropdown compacto

### 2.4 Login (`Login.tsx`)
- Manter layout atual (já responsivo)
- Ajustar padding para tablets

## 3. Componentes de UI

### 3.1 Cards
- Padding consistente: `p-4` (mobile), `p-5` (desktop)
- Border radius consistente: `rounded-lg` (mais consistente)
- Sombras mais sutis em mobile

### 3.2 Tabelas
- Header fixo em scroll vertical
- Células com padding adequado para touch
- Overflow horizontal com scroll suave

### 3.3 Botões
- Tamanho mínimo para touch: `min-h-10`
- Icons com spacing adequado

### 3.4 Formulários
- Labels sempre visíveis
- Inputs com altura consistente
- Validation messages mais visíveis

---

## 4. Breakpoints e Espaçamentos

```css
/* Breakpoints pretendidos */
- Mobile: < 640px
- Tablet: 640px - 1024px  
- Laptop: 1024px - 1440px
- Desktop: > 1440px
```

### Espaçamentos Recomendados
- Container padding: `px-4` (mobile), `px-6` (tablet+)
- Section spacing: `space-y-4` (mobile), `space-y-6` (desktop)
- Card gap: `gap-3` (mobile), `gap-4` (desktop)

---

## 5. Melhorias Visuais Específicas

### Para Laptop (1024px - 1440px)
- Evitar espaços whitespaces excessivos
- Cards com largura variável mas limitada
- Sidebar do mobile mais prática com animações

### Otimizações de Performance
- Lazy load de gráficos
- Menos renderizações desnecessárias
- Animações mais leves
